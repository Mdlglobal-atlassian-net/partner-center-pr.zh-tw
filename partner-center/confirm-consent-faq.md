## <a href="" id="processfaq"></a>Process

**什麼是新的程序，確認 Microsoft Cloud 合約客戶接受？**

Microsoft 會致力於我們的客戶和合作夥伴持續符合規範。 身為 Microsoft 的雲端解決方案提供者 (CSP) 計畫的合作夥伴，您有責任檢閱 Microsoft 的條款及條件，包括 Microsoft 產品及服務與客戶可接受的使用。 更清楚地協助合作夥伴符合合規需求並確保透明度，我們會啟用新的程序需要雲端解決方案提供者合作夥伴，以確認其客戶已接受的 Microsoft Cloud 合約條款。 截至 2018 年 8 月 7，我們已導入新的欄位，在合作夥伴中心儀表板使用者介面也適用於的合作夥伴中心 API 合作夥伴提供這項資訊。 一開始就會選擇性的以提供此確認，但是它將會需要從 2018 年 11 月 7 日開始。 

**這項需求時開始生效？**

合作夥伴確認接受儘速功能都可在 2018 年 8 月 7 日開始。 從 2018 年 11 月 7，合作夥伴中心儀表板使用者必須以確認所有新的交易 Microsoft Cloud 合約客戶接受。 在 2019 年 3 月 22 日合作夥伴中心 API 的使用者必須執行相同。 API 使用者的期限已擴充為允許它們在其他時間技術的實作。

**新 |Microsoft 是否可以驗證由合作夥伴所提供的客戶資訊的正確性？**

Microsoft 會驗證提供的客戶的電子郵件地址是正確的格式。 它是合作夥伴必須負責提供正確的資訊，並更正任何可找到不正確的資訊。 

**新 |如何可以合作夥伴證明 Microsoft Cloud 合約，客戶的接受是否已簽署，記載，數位化，或在紙上？**

合作夥伴負責確保客戶已接受的 Microsoft Cloud 合約條款，以及管理他們自己的處理程序和記錄。 藉由使用合作夥伴中心儀表板或合作夥伴中心 API 的程序，合作夥伴就確認客戶已接受的 Microsoft Cloud 合約條款。

**新 |如何將 Microsoft 檢查確認程序是為客戶完成？**

11 月 7，從 2018 年開始，當雲端解決方案提供者合作夥伴起始交易，為新的或現有客戶使用合作夥伴中心儀表板時，Microsoft 的系統將會自動檢查客戶確認資訊是否出現在合作夥伴中心儀表板。 如果不存在，將無法完成交易。 

開始 3 月 22 2019 年，當雲端解決方案提供者合作夥伴初始交易新的或現有客戶使用合作夥伴中心 API，Microsoft 的系統將會自動檢查客戶確認資訊是否存在。 如果不存在，將無法完成交易。 

**新 |確認程序中提供其電子郵件地址的客戶傳送的通知？**

沒有任何通知處理程序。

## <a href="" id="policyfaq"></a>原則

**Microsoft 要求合作夥伴，以確認客戶已經接受 Microsoft Cloud 合約的為何？** 

Microsoft Cloud 合約保護合作夥伴、 客戶和 Microsoft 藉由提供客戶的使用權限、 條款和條件授與 Microsoft Cloud 的產品和服務的訂單。 

適當執行的 Microsoft Cloud 合約保護我們的合作夥伴，客戶和 Microsoft 藉由確保許多重要的主題包括相互對齊方式，但不是限於安全性、 隱私權及資料保護。 確認程序可確保透明度和對齊方式，並可讓 Microsoft 更清楚地協助客戶需要立即回應法規詢問。

**哪些客戶資訊，才能完成確認程序？**
接受授權合約的人相關，則需要下列資訊：
-   合約接受的日期
-   第一個和最後一個名稱
-   電子郵件地址
-   電話號碼 （選用） 

**新 |如果我們不符合到期日，會發生什麼事？**

合作夥伴確認接受儘速功能都可在 2018 年 8 月 7 日開始。 從 2018 年 11 月 7，合作夥伴中心儀表板使用者必須以確認所有新的交易 Microsoft Cloud 合約客戶接受。 在 2019 年 3 月 22 日合作夥伴中心 API 的使用者必須執行相同。

在強制執行時，直到已確認客戶接受不會允許至座位上現有的訂閱計數的現有客戶或變更的新購買項目。 針對新的客戶，直到已確認接受不會允許交易。 自動續約不會受到這項需求。

**哪些地區沒有這項需求適用於？**

需要 Microsoft 公用雲端底下的所有地區的 Microsoft Cloud 合約客戶接受的確認。 確認客戶接受的不是支援國內雲 （包括適用於美國政府和 Microsoft Cloud 德國的 Microsoft Cloud）。

**哪些合作夥伴負責確認至 Microsoft Cloud 合約客戶接受？**

有效 7th，2018 年 11 月，雲端服務提供者 (CSP) 的間接提供者和雲端解決方案提供者直接帳單合作夥伴負責確認客戶接受而且可能會使用合作夥伴中心 API 或合作夥伴中心儀表板來執行這項操作。 間接提供者可能需要先從他們的間接經銷商收集所需的資訊，然後將它提供給 Microsoft，透過合作夥伴中心 API 或合作夥伴中心儀表板。 Contractually，它是間接提供者提供者合約與 Microsoft 以確保其雲端解決方案提供者經銷商結束客戶已經接受要 Microsoft Cloud 合約的繫結。 

**哪些客戶案例需要確認的 Microsoft Cloud 合約客戶接受？**

當雲端解決方案提供者合作夥伴想要進行下列稅金的交易時，必須確認的 Microsoft Cloud 合約接受：
1.  新的客戶完成新購買項目
2.  現有客戶完成新購買項目
3.  現有客戶更新至現有訂閱的基座計數

**我是否可以確認接受現有客戶不同於其新購買項目或座位新增項目？**

是。 客戶接受可以確認 Microsoft Cloud 合約所涵蓋的所有客戶。

**有多個雲端解決方案提供者合作夥伴的客戶需求為何？**

確認已被接受已由合作夥伴確認。 如果客戶 transacts 與多個雲端解決方案提供者合作夥伴，將需要每個合作夥伴確認接受。

**我是雲端解決方案提供者間接經銷商與雲端解決方案提供者間接提供者合作。 為何我責任？**

Contractually，它是間接提供者提供者合約與 Microsoft 以確保給客戶已接受 Microsoft Cloud 合約的繫結。 不過，間接提供者可能會有不同的排列方式與間接經銷商收集客戶接受。 在此情況下，間接提供者必須第一次從他們的間接經銷商收集所需的客戶接受資訊，然後將它提供給 Microsoft，透過合作夥伴中心 API 或合作夥伴中心儀表板。 您可以藉由立即提供所需的資訊，您的間接提供者在他們的要求時支援這個程序。

**我是雲端解決方案提供者間接提供者。 需要與我的雲端解決方案提供者間接經銷商準備協調嗎？**

需要收集所需的客戶接受資訊從他們的間接經銷商，間接提供者建議通訊立即來協調傳遞與接收的這項資訊到他們的間接經銷商這項需求。

**我如何可以解釋我的客戶，他們應該確認他們的 Microsoft Cloud 合約接受？**

藉由確保許多重要的主題包括相互對齊方式保護 Microsoft、 合作夥伴及客戶的 Microsoft Cloud 合約適當執行，但不是限於安全性、 隱私權及資料保護。 確認程序可確保透明度和對齊方式，並可讓 Microsoft 更清楚地協助客戶需要立即回應法規詢問。

**新 |如果客戶使用單一雲端解決方案提供者間接提供者的運作方式，但與多個雲端解決方案提供者間接經銷商完成下列稅金的交易，確實每個雲端解決方案提供者間接經銷商需要提供雲端解決方案提供者間接提供者確認資訊嗎？**

雲端解決方案提供者間接提供者負責提供給 Microsoft 的 Microsoft Cloud 合約客戶接受的確認。 為每個客戶帳戶，只有一個確認可以錄製合作夥伴中心儀表板中一次。 

雲端解決方案提供者間接提供者負責確認負責一併同意 Microsoft Cloud 合約客戶連絡人已完成的操作，無論他們與之交易的雲端解決方案提供者間接經銷商的數目。

**新 |誰應以確認接受的 Microsoft Cloud 合約客戶組織內已獲授權的人員？**

它是合作夥伴必須負責向客戶組織內部人員取得 Microsoft Cloud 合約客戶接受。 它是以判斷提供接受獲得客戶的責任。 

**新 |哪一個版本的 Microsoft Cloud 合約是合作夥伴負責確認客戶接受？ 合作夥伴需要指出哪一個版本的 Microsoft Cloud 合約客戶已接受嗎？**

合作夥伴負責進行雲端解決方案提供者交易與新客戶時，請使用最新的 Microsoft Cloud 合約。 合作夥伴必須確認接受的最新的 Microsoft Cloud 合約時當天、 變更或完成新與現有客戶的交易。 

在 Microsoft Cloud 合約，在續約之前更新案例中，合作夥伴不需要有客戶重新取得 Microsoft Cloud 合約客戶接受立即，但必須 reconfirm 同意 Microsoft Cloud 合約之前將若要續約。 不過，您可能想讓客戶接受新的詞彙之前續約 Microsoft Cloud 合約利用新的和改進的條款。

**新 |我是雲端解決方案提供者間接經銷商。 如果我在我的公司記錄中記錄的 Microsoft Cloud 合約客戶接受，我需要也提供這項資訊透過合作夥伴中心儀表板或合作夥伴中心 API？**

Contractually，它是間接提供者提供者合約與 Microsoft 以確保給客戶已接受 Microsoft Cloud 合約的繫結。 不過，間接提供者可能會有不同的排列方式與間接經銷商收集客戶接受。 在此情況下，間接提供者必須第一次從他們的間接經銷商收集所需的客戶接受資訊，然後將它提供給 Microsoft，透過合作夥伴中心 API 或合作夥伴中心儀表板。

**新 |如果我的客戶不符合 Microsoft Cloud 合約的條款？**

身為 Microsoft 的雲端解決方案提供者計畫的合作夥伴，您必須負責在之前下訂單的 Microsoft Cloud 合約客戶接受項目。 這不是新的需求。 基於任何原因，如果客戶不會再同意 Microsoft Cloud 合約，然後合作夥伴必須取消此客戶相關聯的訂閱。

**新 |可以轉銷商提供代表客戶同意嗎？**

否。

**一旦我確認客戶已提供接受度，我需要重新確認它客戶在未來？**

一旦您已經確認客戶已接受 Microsoft Cloud 合約、 這項資訊會維持在合作夥伴中心客戶帳戶] 區段中並不需要 reconfirm 接受客戶涵蓋的時間期間合約條款。 您必須只 reconfirm 接受之前放置新訂單。

在 Microsoft Cloud 合約，在續約之前更新案例中，合作夥伴不需要有客戶重新取得 Microsoft Cloud 合約客戶接受立即，但必須 reconfirm 同意 Microsoft Cloud 合約之前將若要續約。 不過，您可能想讓客戶接受新的詞彙之前續約 Microsoft Cloud 合約利用新的和改進的條款。

**新 |我不舒服我結束客戶的直接連絡資訊給 Microsoft。 Microsoft 如何使用這項資訊？**

合作夥伴將會建立客戶租用戶，就需要提供客戶的主要連絡人資訊。 每個線上服務條款，「 Microsoft 將會處理個人資料僅限記載於文件從客戶的指示。 」

## <a href="" id="technicalfaq"></a>技術

**合作夥伴如何確認客戶接受及它提供給 Microsoft？ 哪些工具可用來支援這個程序？**

有兩種方法是從 2018 年 8 月 7 日開始提供，來協助合作夥伴： 
1.  合作夥伴可以使用合作夥伴中心 API 以程式設計方式確認客戶接受。 
2.  合作夥伴可以使用合作夥伴中心儀表板來確認客戶接受。

**新 |我是使用合作夥伴中心 API 的雲端解決方案提供者合作夥伴。 哪些 API 變更我需要考慮嗎？** 

在此情況下，會針對 API 變更：

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent) 

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata](https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata)

-   [https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent)

**如果我有實作合作夥伴中心 API 的難度？**

接受可以透過合作夥伴中心儀表板手動確認。

**新 |使用合作夥伴中心儀表板或合作夥伴中心 API 的替代方法，為可以我大量上傳檔案的客戶確認資訊改為？**

在這個時候，在合作夥伴中心儀表板和合作夥伴中心 API 會接受的只用這種提供的 Microsoft Cloud 合約客戶接受的確認。

**新 |我是使用第三方平台或 「 中介軟體，「 執行功能給我的公司系統聯繫在合作夥伴中心的合作夥伴。 我該怎麼做？**

直接與他們的方案與時間軸相關的提供者說的整合的合作夥伴中心 API 功能，可在客戶確認程序，建議使用整合合作夥伴中心 API 的平台的合作夥伴。

**新 |您可以匯入的客戶的完整名稱、 電子郵件地址和日期的 Microsoft Cloud 合約接受提供 API 流程範例？**

合作夥伴建議檢閱詳細資訊和範例的合作夥伴中心 API 文件。

**新 |如何取得最新的 Microsoft Cloud 合約，向客戶顯示，並確認記錄中使用我的 API 程序的一部分？**

目前的合作夥伴中心 API 是非常基本並不會提供合作夥伴能夠以程式設計方式擷取一份 Microsoft Cloud 合約中，也不會擷取版本的 Microsoft Cloud 合約客戶接受確認的一部分。 

**新 |針對此程序使用合作夥伴中心 API 需要 userID 參數。 訂單透過第三方平台不整合於 Azure AD 時如何管理這項需求？**

UserID 在此情況下會識別下合作夥伴租用戶提供的確認客戶同意的員工。 如果員工沒有對應的 Azure AD 使用者帳戶底下的合作夥伴租用戶，則建議您針對該用途指定特定的使用者帳戶，並建立處理程序分別擷取的員工提供身分識別確認。 

**新 |Microsoft 要求合作夥伴，以確認而不是直接向客戶取得它的 Microsoft Cloud 合約客戶同意的為何？**

從雲端解決方案提供者合作夥伴，不會直接向 Microsoft 購買客戶。 適用於使用服務 Microsoft 提供，Microsoft 提供的名詞，Microsoft Cloud 合約中所述。 Microsoft 雲端轉銷商合約需要有檢視並接受之前下訂單的 Microsoft Cloud 合約客戶的雲端解決方案提供者合作夥伴。 

**新 |我使用 API 和合作夥伴中心儀表板 web 使用者介面以確認 Microsoft Cloud 合約客戶接受。 在 2018 年 11 月 7 年，為何我的選項，以確認客戶接受？**

這兩個選項是可用且可接受確認 Microsoft Cloud 合約客戶接受。 交易使用合作夥伴中心儀表板將會需要確認要有從 2018 年 11 月 7 日開始。 交易使用合作夥伴中心 API 將會需要確認要有開始 2019 年 3 月 22 日。
